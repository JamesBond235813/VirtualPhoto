# 前端重构说明

## v5（2026-06-12）：权限体系 + 注册登录 + 历史页重构

**页面权限**：用户与充值、供应商与定价、支付与财务三页仅管理员可见（导航隐藏 + 视图切换守卫双重拦截，普通用户/未登录强行切换会被弹回案例库）。

**注册 / 登录**：顶栏用户胶囊弹窗升级为「登录 / 注册」双 Tab；注册仅需账号 + 密码（≥6 位，二次确认），成功后自动登录，手机验证码注册预留后续接入。HTML 中预填的管理员账密已移除。未登录点击生成按钮会弹出登录/注册弹窗并附引导提示。

**充值入口（普通用户三处可达）**：① 顶栏头像弹窗「在线充值」② 生成页输入卡右下「余额 ¥x · 充值」胶囊 ③ 历史页「当前余额」卡上的充值按钮。管理员手动调账仍在管理员专属的用户页。

**历史页重构**：顶部三张统计卡（当前余额+充值入口、期间充值合计、期间消费合计）；流水支持按日 / 按周（周一起算）/ 按月折叠分组，组头显示笔数与充值/消费小计，点击展开收起（默认展开最新一组）；支持日期区间查询（服务端过滤，上限 500 条）与重置。后端 `/api/users/:id/wallet` 新增 from/to 参数，新增 `/api/register` 注册接口（重复账号、长度校验）。

## v4（2026-06-11）：支付体系（充值 / 单次付费 / 财务）

新增完整支付能力，前后端与数据库同步扩展：

**支付渠道**：「财务」页（仅管理员可见）配置企业支付宝（当面付扫码）与企业微信支付（Native 扫码），纯 node:crypto 签名实现、无 SDK 依赖；支付结果靠轮询查单确认，本地部署无需公网回调。每个渠道支持「模拟模式」（不填真实商户参数即可走通全流程，扫码弹窗内出现「模拟支付成功」按钮）与「生产模式」（必填参数校验、真实收款）。敏感密钥保存后不回显，留空保存表示沿用。

**用户充值**：登录后用户弹窗内「在线充值」→ 选金额（预设/自定义）与渠道 → 二维码弹窗 → 轮询确认 → 余额自动到账（幂等防重复入账），充值记录进入「历史」页流水。

**单次付费生成**：余额不足的用户点击生成时自动弹出「单次付费」流程：选渠道 → 扫码支付 → 支付成功自动提交生成。订单核销有金额校验与防复用；若生成失败订单不核销，可免费重试。

**站点财务**：财务页展示站点余额（累计收款 − 累计提现）、收支流水（充值/单次付费订单）、提现台账（申请校验余额、标记完成）。注：提现为台账记录，实际打款在商户后台操作。

**结构**：新表 payment_configs / payment_orders / withdrawals（服务启动自动建表，老库免迁移）；新增 server/payments.mjs 与 /api/payments/*、/api/finance/* 路由；新依赖 qrcode（启动器会自动 npm install，缺失时优雅降级）。preview.html 改由 `node scripts/build-preview.mjs` 从 index.html 自动生成。

**测试**：签名（PEM/裸 base64 私钥、支付宝签名串、微信 Authorization 头）4 项单测 + 支付全流程（配置→下单→支付→入账→核销→对账→提现）10 组断言全部通过；服务器无 MySQL 冒烟正常。

## v3.1（2026-06-11）：供应商页面向导化

供应商页推倒重做：删除原先两栏表单 + 三张大表，改为「新建供应商」按钮 + 供应商卡片列表。

新建/编辑走**两步向导弹窗**：第一步填名称、Base URL、API Key；点「下一步」自动调用接口解析可用生图模型与价格，第二步逐行勾选模型、改展示名、改价格（解析不到价格的行有黄色高亮，必须手动定价），可随时「返回上一步」；确认后供应商与所选模型一次性入库。页面卡片只显示名称与模型定价（**不再显示 Base URL 和 Key**），每张卡带启用开关、编辑、删除；编辑时 Key 留空表示沿用原 Key（后端已支持），取消勾选已上架模型即下架。

## v3（2026-06-11）：借鉴 ATokenAPI Studio 风格

应用户要求，充分借鉴参考站（保存于 `参考样本站点/`）的设计语言，前端三件套再次整体重写。后端、数据库、API 依旧零改动。

### 移植了什么

**设计系统**：紫罗兰渐变主色（#7c3aed → #8b5cf6 → #a78bfa），深紫黑三层背景（#0f0d14 / #16131f / #1c1829），重玻璃拟态（20/32px 模糊），紫色渐变滚动条，弹簧动效曲线，**深 / 浅双主题**（侧栏底部一键切换，记住偏好）。

**粒子背景**：26 颗紫色光点缓慢上浮漂移，深色主题 0.6 透明度，浅色 0.3，`prefers-reduced-motion` 时自动关闭。

**对话式创作页**（核心借鉴）：原「创作画布」表单+画布布局推翻，改为参考站的 timeline + composer 模式——

- 上方时间线：每次生成一条记录卡（模型、状态徽章、扣费、时间、Prompt 可展开、结果图可放大/下载/复用 Prompt），登录后自动载入最近 12 条历史
- 底部 composer 玻璃卡：左侧「+」上传参考图（缩略图带移除角标）、大输入框（⌘+Enter 直接生成）、参数胶囊（模型、案例参考、提示词辅助、授权确认）、单价徽章、圆形渐变发送键
- 空状态为脉冲发光方球（emptyOrbPulse 动效）

**布局**：侧边栏改为 88px 紧凑图标式（图标+小字标签、激活态荧光指示条），登录/用户信息改为顶栏胶囊+弹窗，提示词辅助收进弹窗。移动端侧栏自动变为底部标签栏。

### 参考站有、本项目暂缺的能力（需后端扩展，前端未做假按钮）

- AI 提示词优化（需接文本模型接口）
- 多图参考（后端目前只收单张 referenceImage）
- 单次批量多张 / 2K·4K 分辨率直传参数（generateImage 暂不接收 size/n）
- 视频生成、资产库、免费工具页、积分体系（本项目用余额计费）

## 如何查看

仅看设计（不需要数据库）：浏览器直接打开 `preview.html`（模拟数据，不落库）。

正式使用：双击 `启动AI照相馆.command`（自动检查 MySQL → 启动服务 → 打开浏览器），或手动 `npm run dev` 后访问 http://localhost:4177。
